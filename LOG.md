- **Fixed task A2 — the §2.3 / §8 contradiction in `yjs-schema-and-sync.md`** (the second-ranked Stage-1 blocker; a Stage-4 data-integrity risk that must be resolved before sync coding begins). The two sections disagreed on the shape of an annotation's `body`: the schema section (§2.3) specified a Y.Array of plain objects; the decision log (§8 item 1) specified Y.Text. With plain objects, a concurrent "edit this note on two devices" scenario produces silent duplicates (`body.delete(0)` + `body.insert(0, newObj)` merges into a two-entry array with no conflict signal). Resolved in favour of §8's Y.Text decision — the shape with proper CRDT semantics. §2.3 completely rewritten to walk through the failure mode step by step, explain why Y.Map body items + Y.Text value fix it, and spell out the trade-off (visible interleaved text beats phantom duplicates for data integrity). Added a `createTextualBody` factory function to hide the construction and called out a concurrent-edit regression test fixture as required from the annotation-layer package's first commit — so a future refactor can't silently reintroduce the broken shape. §2.2's example code updated to use the new shape. §8 item 1 wording tightened to point at §2.3. JSON-LD wire format is unchanged: Y.Text projects to plain string on export, Y.Map to plain object.
- **Fixed task A3 — wired in the deterministic-build CI check** (the third-ranked Stage-1 blocker). Created `website/.github/scripts/check-determinism.mjs` — a 143-line Node harness that runs `npm run build` twice, walks `dist/`, computes SHA-256 of every file, and diffs the two hash maps. Prints differing paths, short hash prefixes, and a pointer at the common non-determinism sources (timestamps, walk order, JSON key ordering, locale-dependent sort, random IDs) on failure. Exits non-zero on any mismatch. Wired to `website/.github/workflows/determinism.yml` — runs on every PR to `main`, every push to `main`, and via `workflow_dispatch` for ad-hoc manual runs. Node 22, same style as the existing `build.yml` / `validate.yml`. Promoted the bullet mention in `tl-pipeline-integration.md` §6.3 into a new §6.4 documenting the harness, its scope (Eleventy output today; the Layer-1 generators land into the same `dist/` and inherit coverage automatically), and its limitation (within-environment only — cross-host determinism between Netlify and CF Pages is a separate test that'll be wired alongside the parallel deploy). **Note before enforcing:** run `node .github/scripts/check-determinism.mjs` locally once before marking this workflow a *required* status check in branch protection — if the current Eleventy build has latent non-determinism, that'll show up as a failure on the first run and any fix should land before the check starts blocking PRs.
